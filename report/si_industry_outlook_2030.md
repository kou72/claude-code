## エグゼクティブサマリー

国内SI業界はAIの波によって**5年以内に構造的な転換点**を迎える。市場全体は2030年に向けて成長を続けるが、その恩恵は「AIを事業の核に据えた企業」に集中し、従来の人月モデルに依存する2次請けSIerは収益性の悪化と存在意義の希薄化に直面する。

当社（100名・32億円・大手SIer向け2次請け）が2030年の中期計画着地点として目指すべき姿は、**「AIネイティブな技術集団」への転換**である。具体的には、(1) AI活用で高生産性を実証し元請けから戦略的パートナーとして指名され続ける立場を確立すること、(2) 自社のAIケイパビリティを武器に、一部案件においてエンドユーザーへの直接提案が可能な体制を構築すること、の2軸が現実的な着地点として浮かび上がる。

**2030年に向けた最重要アクション（後述で詳述）:**
- AI活用による開発生産性の自社内50%向上（業界水準との同期）
- AI部隊を全社横断的な「AI推進室」として再定義し、全部隊のAI対応を底上げ
- 特定の業種・技術ドメインでの「強み領域」を絞り込み、上流提案力を獲得
- SES型人月収益への依存度を段階的に引き下げ、プロジェクト請負・マネージドサービス比率を高める

## 1. 国内SI業界の現状

### 1-1. 市場規模と成長率

[IDC Japanの調査](https://www.imagazine.co.jp/idc-it-service-market2024/)によると、国内ITサービス市場は2024年時点で**7兆205億円**（前年比+7.4%）と好調を維持している。ただし成長率は鈍化傾向にあり、[日経xTECHのIT主要15社調査](https://xtech.nikkei.com/atcl/nxt/column/18/00848/00164/)では2025年度は+6.3%への鈍化が見込まれる。中長期的には年平均**3.8%成長（CAGR）**に落ち着く見通しだ（IDC、2030年市場規模予測：22.4兆円）。

| 指標 | 数値 |
|------|------|
| 国内ITサービス市場（2024年） | 7兆205億円（前年比+7.4%） |
| IT主要15社サービス売上合計（2024年度） | 14兆5,730億円（前年度比+8.9%） |
| 2025年度成長率見通し | +6.3%（鈍化） |
| 2030年市場規模予測（IDC） | 22.4兆円 |
| 2025〜2030年CAGR | 3.8% |

セグメント別では、コンサルティング＆SI（7.8兆円、+10.1%）が最も高い成長率を示し、クラウドサービス（8,526億円、+19.5%）が急拡大している。一方、従来型マネージドサービス（5.9兆円、+6.0%）は相対的に伸び悩む。

**現在の好調を支える需要要因：**
- 官公庁・自治体システムの大型刷新（マイナンバー・ガバメントクラウド対応）
- 基幹システムのモダナイゼーション（老朽化したレガシーシステムの刷新）
- DX推進への民間投資増加

## 3. AIがSI業界に与えるインパクト

### 3-1. 人月モデルの構造的崩壊

SI業界の収益の根幹である「人月単価 × 工数」モデルが、AIの生産性向上によって**根本から揺らぐ**。

- **TIS（大手SIer）の公式目標：** [2029年度までにシステム開発生産性**50%向上**](https://xtech.nikkei.com/atcl/nxt/column/18/00849/00178/)
- **実際の効果（GitHub Copilot・Accenture共同研究）：** [コーディングタスクを平均**55%速く**完了](https://github.blog/news-insights/research/research-quantifying-github-copilots-impact-in-the-enterprise-with-accenture/)
- **工数削減の影響：** 生産性50%向上 = 同じ成果物を出す工数が実質3分の2に = 顧客から**料金引き下げ要求**が正当化される

この論理は極めてシンプルだ。「100人月 × 70万円 = 7,000万円の案件が、AIで50%効率化されると50人月相当 = 3,500万円への値下げ圧力がかかる」という構造的問題が発生する。これは従来の人月モデルを採用する企業の**売上・収益を直撃する**。

[日経ビジネス](https://business.nikkei.com/atcl/gen/19/00322/102900152/)は「5年もあれば、ウォーターフォール型のシステム開発は生成AIで自動化できる」と指摘。「下請け技術者の大半が無用の存在に化す」という厳しい見方が業界内に広がっている。

### 3-2. AIコーディングツールの急速な普及

AIを活用したコーディング支援は「近未来の話」ではなく、**2026年時点で既に業界標準化しつつある現実**である。

[Second Talentの統計](https://www.secondtalent.com/resources/vibe-coding-statistics/)によれば、AIコーディングツールの採用率は開発者全体の44%（米国では日常利用率92%）、Fortune 500企業では87%に達する。[GitHub Copilotのユーザー数は2025年7月時点で2,000万人超](https://endroid.com/2025/github-copilot-surpasses-20-million-users-eyes-enterprise-ai-coding-dominance/)（Fortune 100の90%が採用）。[Gartnerは2026年末までに新規ソフトウェアコードの60%がAI生成になると予測](https://www.gartner.com/en/newsroom/press-releases/2026-02-03-gartner-forecasts-worldwide-it-spending-to-grow-10-point-8-percent-in-2026-totaling-6-point-15-trillion-dollars)している。

| 指標 | 数値 |
|------|------|
| AIコーディングツール採用率（開発者全体） | 44%（米国では日常利用率92%） |
| Fortune 500企業の採用率 | 87% |
| GitHub Copilotユーザー数（2025年7月） | 2,000万人超（Fortune 100の90%が採用） |
| 全世界のコードに占めるAI生成比率（2024年） | 41%（256億行相当） |
| Gartner予測（2026年末） | 新規ソフトウェアコードの**60%がAI生成** |
| Anthropic社内（Claude Code） | コードの**90%がAI生成**（CEO談） |

中堅SIerでの具体的な導入事例も出てきた。[テックファームがWindsurfを全社標準化したケース（TECH PLAY）](https://techplay.jp/column/2068)では、「異なるスキルレベルのメンバーでも従来困難だった品質水準を実現、上位エンジニアへの相談件数が減少」という効果が報告されている。

### 3-3. エンジニア需要の二極化

AIの普及はエンジニア需要を**均一に減らすのではなく、二極化させる**という点が重要だ。[経済産業省「IT人材需給に関する調査」](https://www.meti.go.jp/policy/it_policy/jinzai/gaiyou.pdf)は以下の見通しを示している。

| エンジニア類型 | 2030年に向けた需要動向 |
|--------------|----------------------|
| **先端IT人材**（AI/ML、クラウド、自動化、上流設計） | 深刻な不足（経産省推計：最大**79万人不足**） |
| **従来型IT人材**（コーディング中心、運用保守） | 余剰方向（同推計：**約10万人余剰**） |

さらに市場での価値格差が広がっており、「AI活用スキルを持つエンジニアと従来型エンジニアの市場価値に**10倍以上の差**が生まれる可能性」も指摘されている。

## 4. 2次請けSIerを取り巻く環境変化

### 4-1. 中小・2次請けSIerの厳しい現実

当社と同規模・同類型のSIerが置かれている環境はすでに厳しい局面に入っており、2024年のデータがその現実を示している。

[帝国データバンクの調査](https://www.tdb.co.jp/report/industry/20250423-software-br24fy/)では、2024年度のソフトウェア業倒産件数は**220件**（前年度154件から1.4倍増、過去10年で初の200件超）。倒産企業の8割以上が従業員10人未満の小規模事業者だ。[東京商工リサーチの調査](https://www.tsr-net.co.jp/data/detail/1200895_1527.html)では情報通信業全体の倒産は2024年に**425件**（前年比+21.7%、11年ぶり400件超）に達している。

| 指標 | 数値 |
|------|------|
| ソフトウェア業倒産件数（2024年度・帝国データバンク） | **220件**（前年度154件から1.4倍増、過去10年で初の200件超） |
| 倒産企業のうち従業員10人未満の割合 | 8割以上 |
| 情報通信業倒産（2024年・東京商工リサーチ） | **425件**（前年比+21.7%、11年ぶり400件超） |
| ソフトウェア業の正社員人手不足率 | **72.2%**（2025年3月時点、全業種トップ） |
| 中小企業の平均営業利益率 | **5.7%**（大手企業11%と比較して半分以下） |
| 中小企業の赤字企業率 | **22.0%**（大手企業4.8%） |

帝国データバンクは2024年の倒産急増の一因として「ユーザー企業の内製化進展」を明示的に挙げている。AIによる内製化加速はすでに中小SIer倒産の現実的な要因になっている。

### 4-2. 多重下請け構造の崩壊リスク

当社が主に立つ「2次請け」のポジションは、以下3つのシナリオから同時に脅威にさらされている。

**シナリオ①：工数削減による単価圧力**
大手SIerが生成AIで生産性を50%向上させると、下流の2次・3次請け企業への発注量が激減する。元請けが「同じ成果を半分の人数で出せる」なら、下請けへの発注規模は縮小する。

**シナリオ②：ユーザー企業の内製化加速**
ローコード/ノーコードツール・AI活用でユーザー企業が自社開発を増やすと、元請けSIer自体の受注が減り、2次請けへの仕事量も連動して減少する。

**シナリオ③：「人集め」機能の陳腐化**
人を集めてマージンを取るSES型ビジネスモデルは、AIが「スキルの底上げ」をした場合に差別化根拠を失う。「100人のエンジニアを集められる」という強みが希薄化する。

### 4-3. 脅威と機会の並存

一方で、以下の**機会要因**も存在する。これらを活かせるかどうかが、当社の2030年着地を分ける。

| 機会要因 | 具体的内容 |
|----------|-----------|
| **DX需要の継続拡大** | 官公庁・地方自治体のシステム刷新需要は2030年まで継続見込み |
| **AI導入支援需要の急増** | 大手SIerがAIサービスを販売増加する中、実装・運用を担う下請けへの需要は短中期的に増加 |
| **レガシーモダナイゼーション** | 既存基幹システムのクラウド移行・AI対応改修は膨大な工数が必要 |
| **セキュリティ・品質保証需要** | AI生成コードの品質保証・セキュリティ検証の需要が新たに発生 |
| **人材不足の逆説** | 先端IT人材不足が続く中、AI活用でレベルアップした当社エンジニアの市場価値は上昇 |

## 5. 2030年・2035年シナリオ分析

### 5-1. 業界全体の2030年シナリオ

Gartner・IDC・経産省の各データを統合すると、業界全体の変化は以下のように推移する見通しだ。[IDCはITサービス契約の30%が2029年までに成果ベースに移行すると予測](https://www.idc.com/resource-center/blog/charting-the-agentic-future-of-it-services-from-projects-to-platforms/)している。[BCGはAIがテクノロジーサービス業界に2,000億ドル規模の新たな機会をもたらす](https://www.bcg.com/publications/2026/the-200-billion-dollar-ai-opportunity-in-tech-services)と分析する一方で、従来型デリバリーモデルの破壊も同時に指摘している。

| 変化の軸 | 2026年（現在） | 2030年予測 | 2035年予測 |
|---------|-------------|-----------|-----------|
| コード生成 | 41%がAI生成 | 60〜90%がAI生成 | AI主導、人間はレビュー・設計中心 |
| SIer契約形態 | 人月モデルが主流 | 成果ベース契約が30%超 | 成果ベース・サブスクが過半数 |
| ITサービス提供方法 | プロジェクト型 | プラットフォーム・モジュール型が台頭 | AIマネージドサービスが主流 |
| 大手SIの収益ドライバー | 実装・運用保守 | AIコンサル・エージェント運用 | AIネイティブサービス |
| 日本クラウド市場規模 | 368億ドル | 約600億ドル（推計） | 808.5億ドル（2031年IDC予測） |
| IT人材不足（日本） | 約30〜40万人 | 最大79万人不足 | 先端/従来型の格差が最大化 |

### 5-2. 当社の3つのシナリオ

**シナリオA：現状維持（高リスク）**
- 現在の2次請けSES中心のビジネスを継続
- 2028〜2030年にかけて単価圧力・発注量減少が顕在化
- 売上は現状維持〜漸減、利益率は悪化
- 2030年時点：売上25〜30億円規模に縮小の可能性

**シナリオB：部分転換（中程度リスク）**
- AI活用で生産性を向上させ、品質・速度で元請けからの信頼を獲得
- 一部案件でエンドユーザーへの直接提案が可能な体制を構築
- SES依存度を下げ、請負・マネージドサービス比率を向上
- 2030年時点：売上40〜50億円、利益率5〜8%

**シナリオC：本格転換（高難度・高リターン）**
- AIを核とした専門サービス企業への転換
- 特定業種（例：製造・金融・公共）でのAIコンサルティング機能を保有
- 人月モデルからプロジェクト請負・マネージドサービスが主収益に
- 2030年時点：売上60〜80億円、利益率10%超

**現実的な着地点はシナリオBを確実に達成しながら、シナリオCの入口に立つことである。**

## 6. 当社への示唆と戦略提言

### 6-1. 2030年の「あるべき姿」の定義

以下の姿を2030年の目標着地点として設定する。

> **「AIネイティブな技術実装集団」として、大手SIerから指名される戦略的パートナーポジションを確立し、一部エンドユーザー直接取引も実現した、売上45〜55億円・利益率7%以上の企業体」**

現在の32億円から45〜55億円への成長は、単純な頭数拡大ではなく「1人当たり生産性の向上」と「ビジネスモデルの一部転換」によって達成する。

### 6-2. 重点戦略（3本柱）

#### 戦略1：AI活用による生産性50%向上と全社底上げ（〜2027年優先）

**なぜ今すぐやるか：** 大手SIerはすでに[「2029年度までに開発生産性50%向上」を掲げている（日経xTECH）](https://xtech.nikkei.com/atcl/nxt/column/18/00849/00178/)。2次請けである当社が生産性向上に乗り遅れると、「AIを使いこなせる下請け」と「使えない下請け」に選別される。前者になれなければ発注量が減る。

**具体的アクション：**
- GitHub Copilot・Cursor等のAIコーディングツールを**全エンジニアに標準装備**（まず1年以内）
- AI部隊を「AI推進室」に格上げし、インフラ・アプリ・Web各部隊のAI活用を横断的に支援する役割を担わせる
- 「AI活用でどれだけ生産性が向上したか」を定期的に計測し、元請けへのアピール材料にする
- [テックファーム事例（工数最大80%削減）](https://techplay.jp/column/2068)を参考に、自社での効果測定を実施

**期待効果：**
- エンジニア1人当たり生産性が向上 → 同じ人数でより多くの案件を受注可能に
- 工数削減による余剰リソースを上流工程・新規開発に振り向ける
- AIスキルが採用市場での差別化要因になる

#### 戦略2：「強み領域」の絞り込みと上流提案力の獲得（〜2028年）

**なぜ絞り込みが必要か：** 「幅広く展開」は現在の強みだが、AIが汎用的なコーディングを代替する時代には「何でもできる」は差別化にならない。特定の業種・技術領域での「深さ」が元請けから指名される根拠になる。[MITスローンの研究](https://mitsloan.mit.edu/ideas-made-to-matter/how-digital-business-models-are-evolving-age-agentic-ai)では、エコシステムドライバー型モデルの採用企業だけが業界平均を上回る収益成長を達成していると指摘している。

**具体的アクション：**
- 既存顧客・案件を分析し、「収益性が高く、かつ拡張できる領域」を1〜2つに絞り込む（例：金融系インフラ×クラウド移行、または製造系アプリ×AI導入支援）
- 絞り込んだ領域での**上流提案（要件定義・アーキテクチャ設計）**ができる人材を育成・採用
- 元請けSIerに対して「この領域なら当社を使うと良い」と想起される専門性を確立
- 可能であれば、当該領域のエンドユーザー企業への直接提案パイロットを開始（2028〜2029年）

**期待効果：**
- 「代替可能な下請け」から「この分野の専門家集団」へのポジション転換
- 単価の向上（専門性プレミアム）
- 一部案件でのエンドユーザー直接取引の実現

#### 戦略3：収益モデルの段階的転換（〜2030年）

**転換の方向性：**

| ビジネスモデル | 現状 | 2030年目標 |
|--------------|------|------------|
| SES（人月派遣型） | 〜70% | 40〜50%に縮小 |
| 請負（プロジェクト型） | 〜30% | 35〜40%に拡大 |
| マネージドサービス・保守 | 〜0% | 10〜20%に新設 |

**マネージドサービスを「0から始める」ことの重要性：** グローバルでは、[AIを活用したMSP（マネージドサービスプロバイダー）がクライアント定着率を30%向上させているデータがある（Forrester）](https://www.futuremarketinsights.com/reports/ai-consulting-services-market)。AIを活用した「継続的な運用・改善支援」は、一度受注すると安定収益になる。インフラ部隊の強みを活かしたAI活用クラウド運用サービスは、当社が狙いやすい領域だ。
