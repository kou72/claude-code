# Claude Code 本社導入ロードマップ

## 導入スコープの比較

**目的：全社員が汎用AIとして活用できる環境を整備する。**

| 軸               | A：汎用AI・本社全体導入              | B：コーディングAI・プロジェクト適用    |
| ---------------- | ------------------------------------ | -------------------------------------- |
| 対象             | 本社全部門                           | 開発プロジェクト単位                   |
| 用途             | 文書作成・調査・分析など汎用         | コーディング・レビュー・設計支援       |
| 顧客調整         | **不要**（顧客データをAIに渡さない） | **必要**（契約上の合意が必要）         |
| 承認難度         | 中（社内ポリシー・セキュリティ）     | 中〜高（社内 + 顧客ごとに個別交渉）    |
| 効果の見えやすさ | 低〜中（個人差が大きい）             | 高（工数・品質で測定しやすい）         |
| 展開スピード     | 中（社内承認が通れば全社一斉）       | 遅い（PJごとに顧客調整が繰り返し発生） |
| 目的との適合     | **高**（全社員活用が目的）           | 低（開発者限定になる）                 |

### 結論

全社員の汎用AI活用を目的とするなら、**A一択**。
Bは顧客調整がボトルネックになるうえ、対象が開発者に限定されるため目的と合わない。

## 用途ごとの制約・ボトルネック整理

### A：汎用AI（文書作成・調査・分析など）

| 用途例               | 制約・ボトルネック                                   | 深刻度                 |
| -------------------- | ---------------------------------------------------- | ---------------------- |
| 社内文書の作成・要約 | 社内機密情報をAIに入力するリスク（情報漏洩ポリシー） | 高                     |
| 調査・情報収集       | ハルシネーション（誤情報）をそのまま使うリスク       | 中                     |
| 会議録・議事録作成   | 録音・文字起こしデータの取扱いポリシー               | 中                     |
| アイデア出し・壁打ち | 制約少ない。リテラシー差による活用格差               | 低                     |
| **導入全体**         | **社内セキュリティポリシーの承認**                   | **最大のボトルネック** |
| **導入全体**         | **活用リテラシーの差（使いこなせない人が出る）**     | 中                     |

### B：コーディングAI（コーディング・レビュー・設計支援）

| 用途例                   | 制約・ボトルネック                                 | 深刻度                 |
| ------------------------ | -------------------------------------------------- | ---------------------- |
| コード生成・補完         | ソースコードをAIに渡すことへの**顧客の合意**が必要 | 高                     |
| コードレビュー支援       | 同上。機密ロジックの漏洩リスク                     | 高                     |
| ドキュメント生成         | コードを含む場合は顧客調整必要                     | 中                     |
| 設計・アーキテクチャ検討 | 設計情報が機密の場合は顧客調整必要                 | 中                     |
| **導入全体**             | **顧客ごとの個別交渉（PJが増えるたびに繰り返し）** | **最大のボトルネック** |
| **導入全体**             | 開発者以外には恩恵がない                           | 高（目的と不一致）     |

### 比較サマリー

|                            | A：汎用AI               | B：コーディングAI            |
| -------------------------- | ----------------------- | ---------------------------- |
| 最大の壁                   | 社内ポリシー承認（1回） | 顧客調整（PJごとに繰り返し） |
| 壁を越えた後               | 全社員が使える          | 該当PJの開発者のみ           |
| 目的（全社員活用）との適合 | 高                      | 低                           |

## A：汎用AI・本社全体導入 ロードマップ

期間目安：約6〜8ヶ月

### Phase 1：現状整理・社内調整（1〜2ヶ月）

#### 情報セキュリティポリシー上の利用可否確認

- 社内セキュリティポリシーの現行文書を収集・確認
- Claude利用時のデータ送信範囲・プライバシーポリシーを調査
- 法務・情報システム部門へのヒアリング実施

#### 本社承認ルート・窓口の確認

- 承認に必要な部門・決裁者のリストアップ
- 稟議フロー・申請書式の確認
- 導入推進の社内担当者（チャンピオン）の設定

#### ライセンス費用・負担部門の確認

- Claude Teamプランの料金・ライセンス体系の調査
- 費用対効果の試算（仮）の作成
- 予算負担部門・計上方法の社内調整

#### 導入スコープ・対象ユーザーの定義

- パイロット対象部門・人数の選定
- 対象外とする業務・データの明確化
- 導入目的・KPIの仮設定

---

### Phase 2：パイロット（2〜3ヶ月）

#### 特定部門・チームで先行利用

- パイロット参加者へのアカウント発行・初期設定
- 利用開始前のキックオフ・オリエンテーション実施
- 活用シーン別のプロンプトサンプル集の提供

#### 活用シーン・効果の収集

- 週次での活用状況アンケート実施
- 活用事例・有効なプロンプトのナレッジ収集
- 定量効果（時間削減等）の記録

#### 課題・リスクの洗い出し

- セキュリティ上の懸念事例の記録・整理
- 活用が進まなかった原因の分析
- 改善点・対策のリストアップ

---

### Phase 3：本社承認申請（1ヶ月）

#### パイロット結果をまとめた稟議資料の作成

- パイロット期間の定量・定性効果のまとめ
- リスク評価と対応策の記載
- 全社展開時のコスト・スケジュール案の作成

#### 本社承認

- 稟議書の提出・説明
- 質疑応答・修正対応
- 承認取得・決裁記録の保管

#### アカウント・ライセンス手配

- 全社員分のライセンス数・契約プランの確定
- 発注・契約締結
- 管理者アカウントの設定・権限管理方針の決定

---

### Phase 4：全社展開（1〜2ヶ月）

#### 利用ガイド・活用事例集の整備

- 利用ルール・禁止事項をまとめたガイドラインの作成
- 部門別活用シーン集の作成（パイロット事例を元に）
- FAQ・問い合わせ対応体制の整備

#### ロールアウト・周知

- 全社向け利用開始アナウンスの発信
- 部門ごとのキックオフ勉強会の実施
- アカウント発行・初期設定サポートの提供

#### 効果測定・継続改善

- 月次・四半期での活用状況レポートの作成
- 活用度の低い部門への個別サポート
- 新機能・活用方法のアップデート共有

---

## B：コーディングAI・プロジェクト適用 ロードマップ

期間目安：初回PJで4〜6ヶ月。PJごとに顧客調整が繰り返し発生。

### Phase 1：対象プロジェクト選定（2週間）

#### 適用候補プロジェクトの選定

- 社内PJ一覧から適用候補の洗い出し
- 開発規模・メンバー意欲・技術スタックの評価
- 優先適用PJの決定

#### 顧客契約上の利用可否を確認

- 既存契約書のAI利用関連条項の確認
- 顧客への事前ヒアリング（調整が必要か否かの判断）
- 適用可否の一覧表作成

---

### Phase 2：顧客調整・契約整備（1〜2ヶ月）← ボトルネック

#### AI活用に関する顧客への説明・合意取得

- 顧客向け説明資料（AI利用範囲・セキュリティ対策）の作成
- 顧客担当者へのプレゼン・質疑応答
- 合意内容の議事録・メモの作成

#### 契約書・覚書への条項追加

- 法務と連携した条項文案の作成
- 顧客との条項内容の交渉・合意
- 契約書・覚書の締結・保管

#### 社内の契約ひな形・チェックリストの整備

- AI利用に関する標準条項のひな形作成
- 顧客調整時のチェックリスト整備
- 法務・営業へのひな形周知

---

### Phase 3：試験適用（1〜2ヶ月）

#### 開発者へのセットアップ・使い方レクチャー

- Claude Code・Cursor等のインストール・初期設定
- 利用ルール・禁止事項の説明
- 基本的な使い方のハンズオン実施

#### コーディング・レビュー・ドキュメント生成で活用

- 日常業務でのAI活用（コード生成・補完・レビュー）
- ドキュメント・コメント生成での活用
- 活用した作業・節約時間の記録

#### 工数・品質への効果を計測

- AI活用前後の工数データの比較
- バグ検出率・コード品質指標の変化の記録
- 開発者からのフィードバック収集

---

### Phase 4：事例化・展開申請（1ヶ月）

#### 効果・課題をまとめた事例レポート作成

- 定量効果（工数削減率・品質改善）のまとめ
- 活用が難しかった場面・改善点の記録
- 顧客への報告資料の作成（必要に応じて）

#### 顧客調整の標準プロセス化

- 今回の調整プロセスの振り返り・改善点の整理
- 他PJでも使えるプロセスフロー・チェックリストの作成
- 法務・営業・PMへの標準プロセスの共有

#### 他プロジェクトへの横展開稟議・本社承認

- 複数PJへの展開計画（対象・スケジュール）の作成
- 稟議書の作成・提出
- 承認取得・展開方針の確定

---

### Phase 5：複数PJ展開（随時）

#### 適用PJを順次拡大

- 各PJでの顧客調整の実施（標準プロセスに従って）
- 開発者へのセットアップ・レクチャーの実施

#### ナレッジ共有・ベストプラクティス整備

- 社内ナレッジベースへの事例・プロンプト集の蓄積
- 定期的な社内勉強会・情報共有セッションの実施

## 自律AIエージェント＋MCP適用可能性検討

### 凡例

| 記号 | 意味                                         |
| ---- | -------------------------------------------- |
| ◎    | 自律AIで完結または大幅自動化が可能           |
| ○    | AIが補助・下書き生成。人の確認・承認が必要   |
| △    | 一部補助は可能だが、主体は人間               |
| ×    | AIでの対応困難（判断・交渉・現場対応が必要） |

想定MCPの例：

- **Notion MCP**：情報管理・ドキュメント読み書き
- **ファイルシステム MCP**：ファイル生成・保存
- **メール MCP**：メール送受信・通知
- **カレンダー MCP**：スケジュール取得・登録
- **Web検索 MCP**：法改正・最新情報の調査
- **勤怠システム MCP**：勤怠データ取得（API連携前提）
- **給与システム MCP**：給与データ取得（API連携前提）
