# Claude Code 本社導入ロードマップ

## 導入スコープの比較

**目的：全社員が汎用AIとして活用できる環境を整備する。**

| 軸 | A：汎用AI・本社全体導入 | B：コーディングAI・プロジェクト適用 |
| -- | ---------------------- | ----------------------------------- |
| 対象 | 本社全部門 | 開発プロジェクト単位 |
| 用途 | 文書作成・調査・分析など汎用 | コーディング・レビュー・設計支援 |
| 顧客調整 | **不要**（顧客データをAIに渡さない） | **必要**（契約上の合意が必要） |
| 承認難度 | 中（社内ポリシー・セキュリティ） | 中〜高（社内 + 顧客ごとに個別交渉） |
| 効果の見えやすさ | 低〜中（個人差が大きい） | 高（工数・品質で測定しやすい） |
| 展開スピード | 中（社内承認が通れば全社一斉） | 遅い（PJごとに顧客調整が繰り返し発生） |
| 目的との適合 | **高**（全社員活用が目的） | 低（開発者限定になる） |

### 結論

全社員の汎用AI活用を目的とするなら、**A一択**。
Bは顧客調整がボトルネックになるうえ、対象が開発者に限定されるため目的と合わない。

---

## A：汎用AI・本社全体導入 ロードマップ

```text
Phase 1：現状整理・社内調整（1〜2ヶ月）
  ├ 情報セキュリティポリシー上の利用可否確認
  ├ 本社承認ルート・窓口の確認
  ├ ライセンス費用・負担部門の確認
  └ 導入スコープ・対象ユーザーの定義

Phase 2：パイロット（2〜3ヶ月）
  ├ 特定部門・チームで先行利用
  ├ 活用シーン・効果の収集
  └ 課題・リスクの洗い出し

Phase 3：本社承認申請（1ヶ月）
  ├ パイロット結果をまとめた稟議資料の作成
  ├ 本社承認
  └ アカウント・ライセンス手配

Phase 4：全社展開（1〜2ヶ月）
  ├ 利用ガイド・活用事例集の整備
  ├ ロールアウト・周知
  └ 効果測定・継続改善
```

---

## B：コーディングAI・プロジェクト適用 ロードマップ

```text
Phase 1：対象プロジェクト選定（2週間）
  ├ 適用候補プロジェクトの選定（開発規模・メンバー意欲）
  └ 顧客契約上の利用可否を確認

Phase 2：顧客調整・契約整備（1〜2ヶ月）  ← ボトルネック
  ├ AI活用に関する顧客への説明・合意取得
  ├ 契約書・覚書への条項追加（利用範囲・データ取扱い）
  └ 社内の契約ひな形・チェックリストの整備

Phase 3：試験適用（1〜2ヶ月）
  ├ 開発者へのセットアップ・使い方レクチャー
  ├ コーディング・レビュー・ドキュメント生成で活用
  └ 工数・品質への効果を計測

Phase 4：事例化・展開申請（1ヶ月）
  ├ 効果・課題をまとめた事例レポート作成
  ├ 顧客調整の標準プロセス化
  ├ 他プロジェクトへの横展開稟議
  └ 本社承認

Phase 5：複数PJ展開（随時）
  ├ 適用PJを順次拡大（各PJで顧客調整を繰り返し実施）
  └ ナレッジ共有・ベストプラクティス整備
```
