# Claude Code vs Copilot：Word / Excel / PowerPoint 活用比較

## まず「どのCopilot」かを整理する

Microsoft Copilotには複数の製品があり、**Word/Excel/PowerPointに統合されているもの**は限られる。

| 製品名 | 概要 | Word/Excel/PPT統合 | 月額（目安） |
| ------ | ---- | :---: | ------ |
| **Microsoft 365 Copilot** | 企業向けMicrosoft 365に統合。リボンから直接呼び出せる | **◎ 深く統合** | $30/ユーザー（Microsoft 365 Business Standard以上が前提） |
| **Copilot Pro** | 個人・家族向けMicrosoft 365用。上記の個人版 | ○ 統合あり | $20/ユーザー |
| **Microsoft Copilot（無料）** | ブラウザ・Windowsサイドバーで使う汎用AI | × なし（コピペのみ） | 無料 |
| **GitHub Copilot** | コーディング特化 | × 対象外 | $10〜/ユーザー |

**以降の比較は「Microsoft 365 Copilot（企業版）」を指す。**

## 結論：どちらを選ぶか

### Microsoft 365 Copilot が向いているケース

- Word・Excel・PowerPointを日常業務の中心で使っている
- ITリテラシーが高くないメンバーも使う（クリックだけで動く）
- SharePoint・Teamsと連携して社内資料を参照させたい
- 個人の業務効率化よりも **組織全体での標準化** を重視する

### Claude Code が向いているケース

- 複雑なVBA・Python処理でOfficeファイルを自動化したい
- Officeは出力先の一つで、**調査・分析・文章生成が主目的**
- コスト重視（1つのサブスクで幅広い用途をカバーしたい）
- プログラミングができるメンバーが中心

### 組み合わせ案

実務上、**両方使う**のが最も効果的。

```text
日常のOffice操作・コンテンツ生成   → Microsoft 365 Copilot
複雑なマクロ・スクリプト生成       → Claude Code
調査・分析・戦略立案・壁打ち       → Claude Code
```

**本社全体に展開するならMicrosoft 365 Copilotが現実的**（Officeを全社員が使っているため）。
Claude Codeは**技術スタッフや推進チームが深く活用**する形が向いている。
