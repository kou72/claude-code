# nginx 設定統一チェックリスト（.88 → .188 相当）

## Phase 1 — 証明書の取得・配置

> CAサーバで以下3ファイルを発行してもらい、.88 に転送する。

| ファイル | 用途 |
| -------- | ---- |
| `server.crt` | サーバ証明書 |
| `server.key` | サーバ秘密鍵 |
| `ca.crt` | CA 証明書（クライアント認証用） |

- [ ] CAサーバで `server.crt` / `server.key` / `ca.crt` を発行
- [ ] 動作確認用クライアント証明書（`client.crt` / `client.key`）も発行
- [ ] 証明書ファイルを .88 に転送

  ```bash
  # 転送先ディレクトリを作成
  sudo mkdir -p /etc/nginx/ssl/ca
  ```

- [ ] 証明書を配置先にコピー

  ```bash
  sudo cp server.crt /etc/nginx/ssl/
  sudo cp server.key /etc/nginx/ssl/
  sudo cp ca.crt     /etc/nginx/ssl/ca/
  sudo chmod 600 /etc/nginx/ssl/server.key
  ```

- [ ] 証明書の検証

  ```bash
  openssl verify -CAfile /etc/nginx/ssl/ca/ca.crt /etc/nginx/ssl/server.crt
  # → server.crt: OK
  ```

## Phase 2 — nginx.conf 更新

- [ ] `/etc/nginx/nginx.conf` を以下の内容に書き換え

  ```nginx
  events {
      worker_connections 1024;
  }

  http {
      access_log /var/log/nginx/access.log;
      error_log  /var/log/nginx/error.log;

      server {
          listen 7273 ssl;
          server_name _;

          ssl_certificate     /etc/nginx/ssl/server.crt;
          ssl_certificate_key /etc/nginx/ssl/server.key;

          ssl_client_certificate /etc/nginx/ssl/ca/ca.crt;
          ssl_verify_client on;
          ssl_verify_depth  2;

          ssl_protocols TLSv1.2 TLSv1.3;
          ssl_ciphers HIGH:!aNULL:!MD5;
          ssl_prefer_server_ciphers on;

          location / {
              proxy_pass http://192.168.11.207;
              proxy_set_header Host              $host;
              proxy_set_header X-Real-IP         $remote_addr;
              proxy_set_header X-Forwarded-For   $proxy_add_x_forwarded_for;
              proxy_set_header X-Forwarded-Proto $scheme;

              proxy_connect_timeout 60s;
              proxy_send_timeout    60s;
              proxy_read_timeout    60s;
          }
      }
  }
  ```

- [ ] 構文テスト（CLI）

  ```bash
  sudo nginx -t
  # → syntax is ok / test is successful
  ```

- [ ] nginx-ui の Config Test でも OK を確認

## Phase 3 — 適用

- [ ] nginx reload

  ```bash
  sudo systemctl reload nginx
  ```

- [ ] nginx-ui でステータスが `running` のままであることを確認
- [ ] nginx-ui のエラーログに異常がないことを確認

## Phase 4 — 動作確認

- [ ] ポート 7273 のリスン確認

  ```bash
  ss -tlnp | grep 7273
  ```

- [ ] クライアント証明書なし → 拒否されることを確認

  ```bash
  curl -k https://192.168.21.88:7273
  # → 400 No required SSL certificate was sent
  ```

- [ ] クライアント証明書あり → 転送されることを確認

  ```bash
  curl -k \
    --cert ~/nginx-ssl/client/client.crt \
    --key  ~/nginx-ssl/client/client.key \
    https://192.168.21.88:7273
  ```

- [ ] proxy_pass 先（`192.168.11.207`）への疎通確認

  ```bash
  curl http://192.168.11.207
  ```
