# 体制ビジネス：構造整理

体制ビジネスの構造を3つの切り口で整理する。

---

## パターンA：登場人物マップ

会社単位と人物単位の2層で整理する。

![パターンA：登場人物マップ](charts/fig1_players.png)

### 会社単位（4社）

| 会社 | 役割 |
| --- | --- |
| エンドユーザー | システムを発注する側。予算・要件・決裁権を持つ |
| プライマリーベンダー | 一次請け。エンドユーザーと直接契約し、全体を管理する |
| 体制企業 | PLの自社。体制まるごとでプライマリーベンダーに入る |
| 要員企業 | SEを供給する会社。体制企業の下に入る |

### 人物単位

| 人物 | 所属 | 役割 |
| --- | --- | --- |
| クライアント（担当者） | エンドユーザー | 要件定義・評価・意思決定 |
| PM（マネージャー） | プライマリーベンダー | 体制依頼・品質管理・クライアント対応 |
| PL（体制リーダー） | 体制企業 | 現場指揮・SE管理・プライマリーベンダー対応 |
| SE（エンジニア） | 要員企業 | 実装・開発・PLの指示を受けて稼働 |
| 各社の営業 | 各社 | 下流→上流への提案・関係維持 |

実線（上→下）は発注・業務の流れ。破線（下→上）は各社営業による提案の流れ。

## パターンB：フェーズタイムライン

「体制ビジネスは時系列でどう動くか」をフェーズで整理する。

![パターンB：フェーズタイムライン](charts/fig2_phases.png)

受注から完了まで4フェーズ。お金が動くのは稼働中だけ（月次請求）。
体制組成フェーズがボトルネックになりやすく、ここのスピードが競争力に直結する。

---

## パターンC：お金の流れ／情報の流れ（分離）

「お金」と「情報」を分けて見ると、構造の非対称性が見えてくる。

### C-1：お金の流れ

![パターンC-1：お金の流れ](charts/fig3_money.png)

体制単価は一括で受け取り、PL稼働分とSE分に分配される。
SE1人増えるごとにマージンが増えるスケーラブル構造。

### C-2：情報の流れ（フェーズ別）

![パターンC-2：情報の流れ](charts/fig4_info.png)

フェーズごとに情報の主役が変わる。
完了フェーズの評価フィードバックはPMからPLへ届くが、その先（体制企業・SE・要員企業）には伝わらない。

## 3パターンから見える問い

| パターン | 問い |
| --- | --- |
| A（登場人物） | 営業担当は自社内か外部エージェントか。リーダーと自社の契約形態は何か |
| B（フェーズ） | 体制組成フェーズをどう短縮するか。完了フェーズの実績をどう次につなぐか |
| C（流れの分離） | 完了後の評価情報はどこに蓄積されるか。お金の分配はどの単位で可視化されているか |
