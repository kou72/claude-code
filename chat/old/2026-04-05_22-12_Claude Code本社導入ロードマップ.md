# Claude Code 本社導入ロードマップ

## 導入スコープの比較

**目的：全社員が汎用AIとして活用できる環境を整備する。**

| 軸 | A：汎用AI・本社全体導入 | B：コーディングAI・プロジェクト適用 |
| -- | ---------------------- | ----------------------------------- |
| 対象 | 本社全部門 | 開発プロジェクト単位 |
| 用途 | 文書作成・調査・分析など汎用 | コーディング・レビュー・設計支援 |
| 顧客調整 | **不要**（顧客データをAIに渡さない） | **必要**（契約上の合意が必要） |
| 承認難度 | 中（社内ポリシー・セキュリティ） | 中〜高（社内 + 顧客ごとに個別交渉） |
| 効果の見えやすさ | 低〜中（個人差が大きい） | 高（工数・品質で測定しやすい） |
| 展開スピード | 中（社内承認が通れば全社一斉） | 遅い（PJごとに顧客調整が繰り返し発生） |
| 目的との適合 | **高**（全社員活用が目的） | 低（開発者限定になる） |

### 結論

全社員の汎用AI活用を目的とするなら、**A一択**。
Bは顧客調整がボトルネックになるうえ、対象が開発者に限定されるため目的と合わない。

## 用途ごとの制約・ボトルネック整理

### A：汎用AI（文書作成・調査・分析など）

| 用途例 | 制約・ボトルネック | 深刻度 |
| ------ | ------------------ | ------ |
| 社内文書の作成・要約 | 社内機密情報をAIに入力するリスク（情報漏洩ポリシー） | 高 |
| 調査・情報収集 | ハルシネーション（誤情報）をそのまま使うリスク | 中 |
| 会議録・議事録作成 | 録音・文字起こしデータの取扱いポリシー | 中 |
| アイデア出し・壁打ち | 制約少ない。リテラシー差による活用格差 | 低 |
| **導入全体** | **社内セキュリティポリシーの承認** | **最大のボトルネック** |
| **導入全体** | **活用リテラシーの差（使いこなせない人が出る）** | 中 |

### B：コーディングAI（コーディング・レビュー・設計支援）

| 用途例 | 制約・ボトルネック | 深刻度 |
| ------ | ------------------ | ------ |
| コード生成・補完 | ソースコードをAIに渡すことへの**顧客の合意**が必要 | 高 |
| コードレビュー支援 | 同上。機密ロジックの漏洩リスク | 高 |
| ドキュメント生成 | コードを含む場合は顧客調整必要 | 中 |
| 設計・アーキテクチャ検討 | 設計情報が機密の場合は顧客調整必要 | 中 |
| **導入全体** | **顧客ごとの個別交渉（PJが増えるたびに繰り返し）** | **最大のボトルネック** |
| **導入全体** | 開発者以外には恩恵がない | 高（目的と不一致） |

### 比較サマリー

| | A：汎用AI | B：コーディングAI |
| - | --------- | ----------------- |
| 最大の壁 | 社内ポリシー承認（1回） | 顧客調整（PJごとに繰り返し） |
| 壁を越えた後 | 全社員が使える | 該当PJの開発者のみ |
| 目的（全社員活用）との適合 | 高 | 低 |

## A：汎用AI・本社全体導入 ロードマップ

```text
Phase 1：現状整理・社内調整（1〜2ヶ月）
  ├ 情報セキュリティポリシー上の利用可否確認
  ├ 本社承認ルート・窓口の確認
  ├ ライセンス費用・負担部門の確認
  └ 導入スコープ・対象ユーザーの定義

Phase 2：パイロット（2〜3ヶ月）
  ├ 特定部門・チームで先行利用
  ├ 活用シーン・効果の収集
  └ 課題・リスクの洗い出し

Phase 3：本社承認申請（1ヶ月）
  ├ パイロット結果をまとめた稟議資料の作成
  ├ 本社承認
  └ アカウント・ライセンス手配

Phase 4：全社展開（1〜2ヶ月）
  ├ 利用ガイド・活用事例集の整備
  ├ ロールアウト・周知
  └ 効果測定・継続改善
```

---

## B：コーディングAI・プロジェクト適用 ロードマップ

```text
Phase 1：対象プロジェクト選定（2週間）
  ├ 適用候補プロジェクトの選定（開発規模・メンバー意欲）
  └ 顧客契約上の利用可否を確認

Phase 2：顧客調整・契約整備（1〜2ヶ月）  ← ボトルネック
  ├ AI活用に関する顧客への説明・合意取得
  ├ 契約書・覚書への条項追加（利用範囲・データ取扱い）
  └ 社内の契約ひな形・チェックリストの整備

Phase 3：試験適用（1〜2ヶ月）
  ├ 開発者へのセットアップ・使い方レクチャー
  ├ コーディング・レビュー・ドキュメント生成で活用
  └ 工数・品質への効果を計測

Phase 4：事例化・展開申請（1ヶ月）
  ├ 効果・課題をまとめた事例レポート作成
  ├ 顧客調整の標準プロセス化
  ├ 他プロジェクトへの横展開稟議
  └ 本社承認

Phase 5：複数PJ展開（随時）
  ├ 適用PJを順次拡大（各PJで顧客調整を繰り返し実施）
  └ ナレッジ共有・ベストプラクティス整備
```

## 間接部門の主要業務（AI活用シーン検討用）

150名規模・SES＋請負の会社として代表的な業務を洗い出す。

### 総務

| 業務 | 内容 |
| ---- | ---- |
| 備品・消耗品管理 | 発注・在庫管理・棚卸し |
| 施設・オフィス管理 | 賃貸契約・設備点検・清掃業者対応 |
| 文書・印鑑管理 | 契約書原本保管・押印申請対応 |
| 社内行事・イベント運営 | 総会・懇親会・社員旅行の企画・手配 |
| 各種届出・許認可管理 | 労働者派遣事業の許可更新など |
| 郵便・宅配・来客対応 | 受付・取次ぎ |

### 労務

| 業務 | 内容 |
| ---- | ---- |
| 勤怠管理 | 打刻集計・残業管理・36協定対応 |
| 給与計算 | 月次給与・賞与計算・控除計算 |
| 社会保険手続き | 入退社・扶養変更・算定基礎届 |
| 有給・休暇管理 | 付与・消化管理・法定基準チェック |
| 就業規則管理 | 改訂・周知・労基署届出 |
| 健康管理 | 健康診断手配・ストレスチェック・産業医連携 |

### 法務

| 業務 | 内容 |
| ---- | ---- |
| 契約書レビュー・作成 | SES契約・請負契約・NDA・業務委託契約 |
| 個人情報保護 | プライバシーポリシー・情報管理規程・インシデント対応 |
| コンプライアンス管理 | 社内規程整備・研修・内部通報対応 |
| 知的財産管理 | 成果物の著作権帰属・契約への反映 |
| クレーム・紛争対応 | 顧客からの異議・損害賠償交渉 |
| **請負特有** | 瑕疵担保・契約不適合対応、検収基準の整備 |

### 人事

| 業務 | 内容 |
| ---- | ---- |
| 採用 | 求人票作成・エージェント対応・面接調整・内定手続き |
| 入退社手続き | オンボーディング・オフボーディング |
| 評価制度運用 | 目標設定・評価シート収集・フィードバック支援 |
| 研修・育成 | 新入社員研修・スキルアップ研修の企画・運営 |
| 異動・配置管理 | 要員の配置調整・スキルマッチング |
| 離職防止・エンゲージメント | 1on1支援・サーベイ実施・フォロー |
