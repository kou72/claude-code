# SESプラットフォーム 設計レポート

**作成日：** 2026-03-10
**前提：** ses_platform_analysis.md の課題分析を踏まえた「打ち手設計」
**本命戦略：** メールをプラットフォームに置き換える。メールの上にツールを乗せると代替性が高くなるため、プラットフォームとしての先行者になることを目指す。

---

## 1. 戦略の根本設計：「メールの代替」ではなく「メールの延長」として入る

### なぜ既存競合は普及しなかったか

Qoala・ようかん・SESコンパス等が普及しない理由は技術の問題ではない。**「メールをやめてプラットフォームを使ってください」という文化変革を求めたから**だ。

これに対して、自社のアプローチは逆転させる：

> **「今のメールはそのままでいい。ただし、そのメールが自動的にプラットフォームのデータになる」**

具体的には、プラットフォーム専用のメールアドレスをユーザーに付与し、スキルシートメールをそこに転送（CCまたはBCC）するだけで自動登録される仕組みを作る。**参加のコストをほぼゼロにする。**

### 最初は「プラットフォーム」を名乗らない

最初の10〜50社への提案は「要員管理SaaS」「スキルシート整理ツール」として行う。

- 「プラットフォーム」を売り込むと「人が集まってから使います」という反応になる
- 「今日から1社でも使える業務効率化ツール」として提案すると導入障壁がゼロになる
- 案件会社が価値を体験してから初めてプラットフォーム化の話をする

---

## 2. 課題への打ち手

### 打ち手①：仲介会社の参加インセンティブ設計

**問題の核心：** 仲介会社は「自社ネットワーク情報を共有すると差別化が消える」と恐れている。

**設計案：フォローグラフ型の情報非対称性を維持する**

フォロー関係を**双方向合意制**にする。仲介会社Aが要員会社Bをフォローしても、Bが承認しない限りAには情報が見えない。既存ネットワークの排他性がプラットフォーム上でも維持される。

```
仲介会社A → フォロー申請 → 要員会社B（承認 or 拒否）
承認されたフォロワーのみ、Bのスキルシート・空き状況が見える
```

加えて、**先行者優位の設計**：
- 早く参加した仲介会社ほど「フォロワー枠の優先申請権」を持つ
- 「参加が遅れると、良い要員会社はすでに他社にフォローされている」という競争圧力
- 中抜き防止：仲介会社の許可なく案件会社が要員に直接オファーできない設計

**なぜ仲介会社が参加すると「得」になるか：**

| プラットフォーム内 | プラットフォーム外 |
| --- | --- |
| AIスコア付きで提案速度が上がる | スキルシートを人力で読む |
| 空き状況がリアルタイムでわかる | 送ってきたタイミングしかわからない |
| 自社の成約実績が蓄積・可視化 | 実績は口頭・メールのみ |
| 先行者として良い要員会社を囲える | 遅れると良い要員は取られる |

---

### 打ち手②：プラットフォーム外逃走の防止

**問題の核心：** 「つながった後に外で直接契約する」問題は技術的に完全には防げない。しかし「逃走コスト」を「逃走メリット（面談費の節約）」より大きくすれば防げる。

**3層の防御設計：**

**層1：実績ロック（データの重力）**
- プラットフォーム経由の案件 → 参画実績が自動記録
- プラットフォーム外の案件 → 自己申告不可、実績として残らない
- 評価・継続率・稼働実績がプラットフォームに蓄積 → 離脱すると資産が消える

**層2：業務フローのロック（Slackモデル）**
- スキルシートの版管理・面談ログ・評価・契約書生成をプラットフォーム上で行う
- プラットフォームを離れると「業務が不便になる」状態を作る

**層3：規約によるセーフガード**
- 「プラットフォーム経由で知り合った後24ヶ月以内の直接契約には紹介料15〜20%を支払う義務」
- 面談費が3,000〜5,000円程度であれば、規約リスクを犯してまで逃走する動機は限定的

---

### 打ち手③：評価データの信頼性確保

**問題の核心：** 取引継続関係では本音の低評価がつかない。評価が形骸化する。

**3つの設計で対処：**

**設計1：評価タイミングの設計**
```
参画終了 → 評価ウィンドウ（14日間）→ 評価入力後に次案件の応募機能が解放
```
「取引終了後・次案件決定前」というタイミングで評価を求める。関係継続への配慮が薄れた状態で入力させる（行動経済学の現在バイアスの逆用）。

**設計2：客観データによる補完**

| データ | 取得方法 | 評価への反映 |
| --- | --- | --- |
| 参画継続率 | 契約更新記録から自動算出 | 「平均参画期間Xヶ月」として表示 |
| 再取引率 | 同じ組み合わせの再成約回数 | 「この仲介会社経由の継続率X%」 |
| 応答速度 | プラットフォーム内のメッセージ応答時間 | 「平均応答時間X時間」として表示 |

主観評価が歪んでいても、客観データが実態を補正する。

**設計3：評価インフレへの対策**
- 相対評価（「同カテゴリ内での順位」を公開）
- 「評価しないコスト」：評価をサボると応募できる案件数の上限が下がる

---

### 打ち手④：「プラットフォームでしか得られない体験」の設計

これが最重要。既存競合との本質的な差別化。

**必須機能（メールでは物理的に不可能なもの）：**

**機能1：スキルシートの構造化データ化**
- Word/PDFのスキルシートをAIが解析し、構造化データに変換
- 「Java 5年以上・金融系・首都圏」での横断検索が可能
- PMが50枚のスキルシートを開かずに候補者を5人に絞れる
- **これはメールでは絶対に実現できない体験**

**機能2：空き状況のリアルタイム表示**
- メールでは「送ってきたタイミングが空き期間」しかわからない
- プラットフォームなら「来月空き」の要員一覧が常時更新
- タイムラグという競争劣位をなくす

**機能3：面談調整の自動化**
- 日程調整メールは1件につき5〜10往復。これをカレンダー連携で自動化
- PMの工数を大幅に削減する

**機能4：マッチングスコア**
- 案件を登録するとマッチ度の高い要員が自動で並ぶ
- 案件会社は「今朝の案件に合う要員Top5」を毎朝受け取れる

## 3. ビジネスモデル設計

### コア課金：「面談成立課金」を軸にする

**なぜ面談成立時か：**
- 逃走防止が最も容易（面談の連絡自体がプラットフォーム経由）
- Indeedの「クリック課金」と同じ「価値創出前払い」の論法
- 金額が小さく（3,000〜5,000円）、導入抵抗が低い

### 課金テーブル

| 課金項目 | 課金対象 | 金額 | 根拠 |
| --- | --- | --- | --- |
| 面談セッティング費 | 案件会社 | 3,000〜5,000円/回（月4回超分） | PM工数0.5h×時給6,000円より安い |
| 案件閲覧プラン（スタンダード） | 要員・仲介会社 | 月9,800円 | 案件情報へのアクセス権 |
| 案件閲覧プラン（プレミアム） | 要員・仲介会社 | 月29,800円 | AI検索・優先提案機能込み |
| 評価・実績データ（アドオン） | 案件会社 | 月9,800円 | フェーズ2以降 |
| 優先表示 | 案件会社 | 月19,800〜49,800円 | フェーズ2以降 |

### フェーズ別課金設計

**フェーズ1（0〜12ヶ月・〜300社）：完全無料期間**

目標は「収益」ではなく：
1. 両側ユーザーの臨界点到達
2. 支払い経路の開通（クレカ登録だけ完了させる）

面談は月4回まで無料。それ以上は有料の入口として機能させる。

**フェーズ2（12〜30ヶ月・300〜1,000社）：面談課金主軸**

```
案件会社200社 × 平均面談8回/月 × 超過分5回 × 3,000円 = 300万円/月
要員・仲介会社300社 × 平均月15,000円 = 450万円/月
合計：約750万円/月（年間約9,000万円）
```

**フェーズ3（30ヶ月以降・1,000社超）：データ資産の収益化**

- 評価・実績データ閲覧アドオン
- APIアクセス（大手企業向け月98,000円〜）
- 採用転換支援（SES→正社員転換の成功報酬）

## 4. 参入・グロース戦略

### 臨界点の設計

同一領域・同一地域に絞った **案件会社10社・要員50社** が最小臨界点。「全国・全言語」で薄く集めるより、「首都圏・Web系Java」で密に集める方が価値が高い。

### 段階別獲得設計

**ステップ1（0→10社）：「メール管理SaaS」として足で稼ぐ**

ターゲット：中堅二次請け（従業員50〜200名規模）のPM・役員
理由：メール地獄の痛みが最も大きく、既存パートナーへの縛りが弱い

アクション：
1. 創業者が直接訪問（Web商談ではなく対面）
2. 「プラットフォーム」の話をしない。「スキルシートの管理どうされてますか？」から入る
3. 「無料で使ってください。1ヶ月後に正直な感想を聞かせてください」
4. 導入後は週1回の定期連絡で課題を即日対応（顧客ではなく共同開発者として関与）

10社の選定基準：
- 業界でのリスペクトが高い会社（後の口コミ効果）
- Web系/インフラ系に強い（案件品質の担保）
- 経営者・上位役職者が「使ってみよう」と言える

**ステップ2（10→100社）：実績を武器に仲介会社を引き込む**

10社の案件会社実績を使って仲介会社へのアプローチ開始。
「○○社の案件が集まっている場所で提案できます」という動機を作る。

10社の実績から作る「説得材料」：
- 「○社の営業担当が週○時間の工数削減」
- 「スキルシート受信から提案まで○分」
- 導入した会社のPMのビデオコメント

**ステップ3（100→1000社）：ネットワーク効果の自走**

自走条件：「プラットフォームに入らないと取引機会を逃す」という感覚が業界に広まること。

具体的な自走設計：
- 「○○社の案件はここにしか出ない」という独自案件の存在
- 「業界でよく知られる企業が使っている」という事実を意図的に可視化
- 成約速度の差別化（メールより○倍速い実績を公開）

### 品質管理：「負け犬プール」を防ぐ

最初の50社は完全招待制。Facebookがハーバード生に絞ったのと同じ原理。

- 既存10社に「各社3社まで招待権」を付与
- 招待した会社の行動品質（成約率・レスポンス速度）を計測して招待者にフィードバック
- 案件・要員に掲載期限を設ける（古い在庫の滞留防止）

## 5. プラットフォームが自己強化するループ設計

```
【案件会社が参加】
        ↓
【スキルシートが構造化されAIスコアが機能し始める】
        ↓
【マッチング速度・精度が上がる】
        ↓
【案件会社の工数削減が実感できる → 継続利用】
        ↓
【仲介会社・要員会社が「使わないと案件を逃す」と感じる】
        ↓
【さらに多くの参加者が増える】
        ↓
【評価・実績データが蓄積され、信頼スコアが機能し始める】
        ↓
【スコアの高い仲介・要員が優先推薦される】
        ↓
【プラットフォームに居続けることのメリットが増大する】
        ↓
【離脱コストが上昇する（実績・評価・ネットワークが消える）】
```

3つのロック：
- **実績ロック**：蓄積した参画実績・評価スコアは移転不可（やめると消える）
- **ネットワークロック**：フォロー関係・承認済みパートナー関係はプラットフォーム内のみ
- **機能ロック**：蓄積データが多いほど推薦精度が上がり、快適さが増す

## 6. 「信頼」をプラットフォームに移植する方法

SES業界の信頼は「長年の付き合い」が担保している。プラットフォームは「長年の付き合いなしに信頼を生成するシステム」を作る必要がある。

**3層の信頼システム：**

| 層 | 内容 | 実装方法 |
| --- | --- | --- |
| 本人確認層 | 「この会社は実在する」 | 法人番号・代表者確認 |
| 実績層 | 「信頼できる取引相手だ」 | 参画実績・継続率・評価スコア |
| 保証層 | 「万が一の時も安心だ」 | スキルミスマッチ保証・途中退場時の代替要員紹介 |

**信頼の移植戦略：**

新規登録時に「すでに一緒に仕事をしたことがある会社・担当者を招待する」機能を設ける（LinkedInの招待メカニズム）。既存の信頼関係をゼロから作り直すのではなく、**すでに信頼している相手とのやり取りをプラットフォームに移す**ことを最初の体験として設計する。

**実績の段階的可視化：**

- フェーズ1：取引実績数のみ（「取引実績15件」）
- フェーズ2：カテゴリ別実績（「大手SIer発注案件7件」）
- フェーズ3：匿名評価の集計（「総合評価4.2/5.0」）
- フェーズ4：プロジェクト種別実績（「インフラ8件、開発7件」）

## 7. 設計全体のまとめ

### 問題と打ち手の対応表

| 前回分析で判明した問題 | 打ち手 |
| --- | --- |
| 仲介会社に参加動機がない | 双方向合意フォロー + 先行者優位 + 認定パートナー特権 |
| プラットフォーム外逃走 | 実績ロック + 業務フローロック + 規約（24ヶ月紹介料） |
| 評価データが信頼できない | タイミング設計 + 客観データ補完 + 評価インフレ防止 |
| 鶏卵問題 | 案件会社先行 + メール転送ボット（参加コストゼロ） |
| マネタイズ構造が未定義 | 面談成立課金（フェーズ1）→月額サブスク（フェーズ2）→データ課金（フェーズ3） |
| 「負け犬プール」リスク | 招待制 + 品質スコアリング + 掲載期限設計 |

### 今すぐやること（プロトタイプ前に必ずやること）

1. **「プラットフォーム」を名乗らない状態で、案件会社10社に「スキルシート管理ツール」として無料提供する**
2. その10社の工数削減・成約改善を数値で記録する
3. 10社の実績をもとに「仲介会社への価値提案」を作る

### 賭けを一つ選ぶとしたら

**「メール転送1設定で参加完了」設計 × 案件会社先行**

理由：他の全ての設計は「ユーザーが増えてから機能する」ものだが、これだけが「参加コストゼロ」というユニークな突破口になる。メール転送ボットによって、既存の業務を変えずにデータが蓄積し始める仕組みは、既存競合が取っていないアプローチであり、かつプラットフォーム移行の最大障壁（慣習変更コスト）を無効化できる唯一の設計である。
