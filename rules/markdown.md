# Markdown 記述ルール

VSCode の markdownlint 拡張（`davidanson.vscode-markdownlint`）に合わせたルール。
`.markdownlint.json` で MD013（行長制限）と MD024（重複見出し）の一部を無効化済み。

## 守るべきルール

### 見出し前後に空行（MD022）

見出しの前後には必ず空行を入れる。

- NG：見出しの直前・直後にテキストが続く
- OK：見出しの前後に1行空ける

### リスト前後に空行（MD032）

リストの前後には必ず空行を入れる。

- NG：テキストの直後にリストが続く
- OK：テキストとリストの間に1行空ける

### コードブロック（MD031 / MD040）

コードブロックの前後に空行を入れ、必ず言語を指定する。
言語が不明な場合は `text` を使う。

### 強調を見出し代わりに使わない（MD036）

行全体が `**太字**` や `*イタリック*` だけの行は警告が出る。
見出しとして使いたい場合は `####` を使う。

- NG：`**重要な考え方：XXX**` だけの行
- OK：`#### 重要な考え方：XXX`

### 末尾改行（MD047）

ファイルは必ず改行で終わる。
Bash コマンドで作成したファイルは末尾改行がない場合があるので注意。

## Markdownファイル作成・編集後の確認（必須）

Markdownファイルを作成・編集したら、必ず以下のコマンドでエラーを確認する。

```bash
npx markdownlint-cli <ファイルパス>
```

エラーが出た場合は全て修正してから完了とする。エラーなし（出力なし）になるまで繰り返す。

## 無効化しているルール

| ルール | 理由 |
| ------ | ---- |
| MD013（行長80文字） | 日本語文書では1文が80文字を超えることが多く非現実的なため |
| MD024（重複見出し）`siblings_only: true` | 打ち手・プレイヤーごとに同じ構造を繰り返す文書では、親見出しが異なれば同名の子見出しを許容する |
