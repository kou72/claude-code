# チャット出力ルール

## 図解が必要な回答の出力方法

テキストだけでは理解負荷が高くなる場面（構造・フロー・比較など）では、
マークダウンファイルに回答を書き出し、図を埋め込んで提示する。

## ファイル管理ルール

- メインファイルはセクション（話題）ごとに **`chat/テーマ名.md`** の名前で管理する
- 同じセクションの回答は同じファイルを上書き更新する
- VSCode のプレビューを開きっぱなしにすることで、更新のたびに自動反映される
- **ファイルを更新する前に**、必ず旧バージョンを `chat/old/` にコピーして履歴を残す

### ファイル名規則

```text
chat/テーマ名.md                          # メインファイル（例: 健康的な食生活.md）
chat/old/YYYY-MM-DD_HH-MM_テーマ名.md    # アーカイブ
```

### ディレクトリ構成

```text
chat/
  テーマ名.md         # セクションごとの最新回答
  charts/             # 生成した PNG 画像
    figN_名称.png
  old/                # 過去の回答の履歴
    YYYY-MM-DD_HH-MM_テーマ名.md
```

## ワークフロー

### 初回（該当テーマのファイルが存在しない場合）

1. PNG を生成（`rules/diagram.md` の手順に従う）
2. Read ツールで生成した画像を確認する（必須）
3. `chat/テーマ名.md` を作成して回答を書き込む
4. ユーザーに案内する

### 更新（該当テーマのファイルが既に存在する場合）

1. PNG を生成（`rules/diagram.md` の手順に従う）
2. Read ツールで生成した画像を確認する（必須）
3. **現在の `chat/テーマ名.md` を `chat/old/YYYY-MM-DD_HH-MM_テーマ名.md` にコピーする**
4. `chat/テーマ名.md` を新しい内容で上書きする
5. ユーザーに案内する（初回のみ）

### 画像確認チェックリスト

- ファイルサイズが 5,000 bytes 以上か
- ラベルが切れていないか・文字化けしていないか
- 矢印の向きや構造が意図通りか
- NG があれば修正して再生成（最大3回）

## ユーザーへの案内文（初回のみ）

```text
`chat/テーマ名.md` を VSCode で開き、
Ctrl+Shift+V（Mac: Cmd+Shift+V）でプレビューしてください。
同じセクションの更新は同じファイルに反映されるので開きっぱなしでOKです。
```

## 判断基準：図解が有効な場面

- フローや手順（ステップの順序・分岐がある）
- 構造・関係（複数の要素の関係性）
- 比較（複数の選択肢・軸の対比）
- プロセス全体図（ステップが4つ以上）

上記に該当しない場合は通常のテキスト回答でよい。
