# チャット出力ルール

## 図解が必要な回答の出力方法

テキストだけでは理解負荷が高くなる場面（構造・フロー・比較など）では、
マークダウンファイルに回答を書き出し、図を埋め込んで提示する。

## ワークフロー

1. PNG を生成（`rules/diagram.md` の手順に従う）
2. **Read ツールで生成した画像を確認する**（必須）
   - ファイルサイズが 5,000 bytes 以上か
   - ラベルが切れていないか・文字化けしていないか
   - 矢印の向きや構造が意図通りか
   - NG があれば修正して再生成（最大3回）
3. `chat/` ディレクトリにマークダウンファイルを作成
4. ファイル内に回答テキストと図を埋め込む
5. ユーザーに「`chat/ファイル名.md` をプレビューで開いてください」と案内する

## ディレクトリ構成

```text
chat/
  YYYY-MM-DD_テーマ名.md   # 回答マークダウン
  charts/                  # 生成した PNG 画像
    figN_名称.png
```

## ユーザーへの案内文

```text
`chat/YYYY-MM-DD_テーマ名.md` を VSCode で開き、
Ctrl+Shift+V（Mac: Cmd+Shift+V）でプレビューしてください。
```

## 判断基準：図解が有効な場面

- フローや手順（ステップの順序・分岐がある）
- 構造・関係（複数の要素の関係性）
- 比較（複数の選択肢・軸の対比）
- プロセス全体図（ステップが4つ以上）

上記に該当しない場合は通常のテキスト回答でよい。
