# 週次プランニング

毎週日曜に実施する。`tasks/backlog.md` を読み込み、Notionを更新して今週のタスクを確定する。

## 前提

- Notionデータベース ID: `3240644e-bfb8-810b-a3fd-d43199e95d04`
- Notion APIトークン: `.mcp.json` 参照
- タスクサイズ基準: S = 30分 / M = 60分 / L = 要分割
- 週タグ形式: `3/16週`（その週の月曜日の日付）
- Notion操作は curl + Unicode エスケープで行う（MCP経由だと文字化けする）

## 手順

### 0. 追加タスクのヒアリング（必須・最初に行う）

プランニング開始時に必ず以下を確認する：

> 「バックログに追加したいタスクはありますか？」

ユーザーが追加タスクを伝えた場合は、以下の順で処理してから Step 1 へ進む：

1. 各タスクの内容・期限・サイズ・カテゴリ・依存関係を整理する
2. `tasks/backlog.md` の適切なセクションに追記する（新セクションが必要なら作る）
3. Notionに登録する（以下の順で行う）：
   a. セクション（プロジェクト）に対応する**親タスクページを先に作成**する
   b. サブタスクページを作成する
   c. サブタスクの `親アイテム` フィールドに親タスクのIDをセットして紐づける
4. 今週着手するものは 週フィールドも設定する

#### 親子関係の設定コマンド

```bash
# サブタスクの親アイテムを設定する
cat > /tmp/link_parent.json << EOF
{"properties":{"\u89aa\u30a2\u30a4\u30c6\u30e0":{"relation":[{"id":"PARENT_PAGE_ID"}]}}}
EOF

curl -s -X PATCH "https://api.notion.com/v1/pages/SUBTASK_PAGE_ID" \
  -H "Authorization: Bearer {TOKEN}" \
  -H "Notion-Version: 2022-06-28" \
  -H "Content-Type: application/json" \
  --data-binary @/tmp/link_parent.json
```

- 親アイテムをセットすると、Notionが自動で親側の `サブアイテム` にも反映する
- サブタスクが複数ある場合は1件ずつ PATCH する

追加タスクがない場合はそのまま Step 1 へ。

### 1. バックログ確認

`tasks/backlog.md` を読み、以下を把握する。

- 各タスクの期限・サイズ・カテゴリ
- 依存関係（`※〇〇後` の記載があるもの）
- 「いつかやる」は今週に入れない

### 2. 週割り当て方針

- **今週（W+0）**: 期限が今週 or 来週のもの、依存なしで着手できるもの
- **来週（W+1）**: 今週タスクの後続・期限が来週末のもの
- **それ以降**: 依存関係があって先行タスクが終わらないと着手できないもの
- 土日の予定・祝日を事前に確認してキャパを調整する
- S×6〜8 + M×1〜2 が1週間の目安ボリューム

### 2.5. スケジュール整合性チェック（必須）

週割り当てが決まったら、Notion更新の前に以下を確認する。
問題があれば週割り当てを修正してから次へ進む。

#### チェック項目

- [ ] **発表・締め切りが週初（月〜火）にあるタスク**：前の週（W-1）中に準備タスクが完了できるか
  - 例：3/30（月）が発表なら、準備タスクはすべて3/29（日）までに完了する必要がある
  - NG：発表と同じ週に準備タスクが割り当てられている
- [ ] **依存関係（`※〇〇後`）の解消確認**：先行タスクが完了していないのに後続を同じ週に入れていないか
  - 先行タスクが完了済みなら `※〇〇後` の注記を削除する
- [ ] **ボリューム過多**：今週のタスク合計が S×8 + M×2 を大幅に超えていないか
  - 超えている場合は優先度の低いものを来週以降にずらす
- [ ] **期限切れタスクの存在**：今日より過去の期限が設定されたタスクが残っていないか
  - 残っている場合は期限を更新するか、完了・削除する

### 3. Notion更新（週フィールドの設定）

```bash
# 週フィールドを更新する（文字化け防止のためファイル経由でJSONを渡す）
cat > /tmp/week.json << 'EOF'
{"properties":{"\u9031":{"select":{"name":"3/16\u9031"}}}}
EOF

curl -s -X PATCH "https://api.notion.com/v1/pages/{PAGE_ID}" \
  -H "Authorization: Bearer {TOKEN}" \
  -H "Notion-Version: 2022-06-28" \
  -H "Content-Type: application/json" \
  --data-binary @/tmp/week.json
```

よく使う週タグと月曜日の対応：

| 週タグ | 月曜日 |
| ------ | ------ |
| 3/16週 | 3/16   |
| 3/23週 | 3/23   |
| 3/30週 | 3/30   |

### 4. タスク完了・削除

```bash
# ステータスを完了にする
cat > /tmp/done.json << 'EOF'
{"\u30b9\u30c6\u30fc\u30bf\u30b9":{"select":{"name":"\u5b8c\u4e86"}}}
EOF

# ゴミ箱に移す
curl -s -X PATCH "https://api.notion.com/v1/pages/{PAGE_ID}" \
  ... --data-binary '{"in_trash":true}'
```

### 5. 印刷用リスト更新

`tasks/backlog.md` の `## 今週の印刷用` セクションを更新する。

- 横35文字を目安に折り返す（全角=2、半角=1）
- **カテゴリ（仕事/学習/個人）ではなく、バックログのセクション（プロジェクト）ごとに見出しを立てる**
- セクション名を見出しにして、タスクを字下げ（`　`）
- 期限があるものは `(～3/23)` 形式で添える
- 完了済みのタスクは外し、新たに今週に入れたタスクを追加する

#### フォーマット例

```text
▢週次プランニング(個人)
プロジェクトA
　▢タスクA(～3/23) ▢タスクB
プロジェクトB
　▢タスクC ▢タスクD
```

## Notionデータベース構造

| プロパティ | 種別 | 備考 |
| ---------- | ---- | ---- |
| タスク名 | title | |
| ステータス | select | 未着手／進行中／完了 |
| カテゴリ | select | 仕事／学習／個人 |
| サイズ | select | S(30分)／M(60分) |
| 期限 | date | |
| 週 | select | 3/16週 形式 |
| 親アイテム | relation | サブタスクの親 |
| サブアイテム | relation | 親タスクの子一覧 |
| 実際にかかった時間 | select | 30分／60分／90分／120分／3時間 |

## 注意事項

- 日付の一対一対応はしない。優先順に上から消化する
- 依存関係があるタスク（`※〇〇後`）は先行タスクの完了前に着手しない
- サブタスクに週を設定する場合、親タスクの週は「完了予定週」にする
- 完了後は「実際にかかった時間」を記録してサイズ見積もりのチューニングに使う
